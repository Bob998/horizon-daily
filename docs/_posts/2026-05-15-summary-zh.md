## [首个公开的 Apple M5 macOS 内核漏洞披露](https://blog.calif.io/p/first-public-kernel-memory-corruption) ⭐️ 8.0/10

Calif 团队披露了首个针对 Apple M5 芯片的公开 macOS 内核内存损坏漏洞，其潜在漏洞赏金高达 150 万美元。 该漏洞标志着 Apple 最新 M5 芯片的安全里程碑，可能影响 Apple 的声誉及更广泛的网络安全格局。同时凸显了漏洞赏金计划在漏洞发现中日益重要的作用。 该漏洞在一份 55 页的报告中详细描述，社区讨论认为其价值在 10 万至 150 万美元之间，具体取决于如何包装。该漏洞据称绕过了 MTE（内存标记扩展），引发了技术上的好奇。

hackernews · quadrige · May 14, 18:25 · [社区讨论](https://news.ycombinator.com/item?id=48139219)

**背景**: 内核内存损坏漏洞允许攻击者破坏操作系统的核心内存，可能导致系统完全被攻破。Apple M5 芯片包含 MTE（内存标记扩展）等安全功能，旨在防止此类漏洞。漏洞赏金计划奖励道德黑客负责任地披露漏洞，Apple 的计划为关键内核漏洞提供高达 150 万美元的赏金。这是首个针对 M5 芯片的公开漏洞，使其成为网络安全领域的重要事件。

## [新 Nginx 漏洞披露，已有缓解措施](https://github.com/DepthFirstDisclosures/Nginx-Rift) ⭐️ 6.0/10

DepthFirstDisclosures 披露了一个名为“Nginx Rift”的新 Nginx 漏洞，通过 2008 年引入的堆缓冲区溢出实现远程代码执行。概念验证需要禁用 ASLR，但研究人员声称存在可靠的 ASLR 绕过方法。 该漏洞影响广泛使用的 Web 服务器，可能波及数百万网站和关键基础设施。尽管已有缓解措施，但 ASLR 被绕过的可能性引发了安全团队和科技基础设施投资者的担忧。 该漏洞需要特定的 Nginx 配置：在替换字符串中包含问号的“rewrite”指令，随后是引用正则捕获组的“set”指令。F5 已修补 1.31.0 和 1.30.1 版本，并建议使用命名捕获而非未命名捕获作为缓解措施。

hackernews · hetsaraiya · May 14, 17:17 · [社区讨论](https://news.ycombinator.com/item?id=48138268)

**背景**: Nginx 是一种高性能 Web 服务器和反向代理，全球超过 30%的网站使用它。ASLR（地址空间布局随机化）是一种通过随机化内存地址来增加利用难度的安全技术。该漏洞源于 2008 年引入的 Nginx 重写模块中的堆缓冲区溢出。

## [Bun 从 Zig 重写为 Rust 已合并](https://github.com/oven-sh/bun/pull/30412) ⭐️ 6.0/10

JavaScript 运行时 Bun 已合并了一个拉取请求，将其核心从 Zig 重写为 Rust，现在包含超过 100 万行 Rust 代码。 此次重写标志着 Bun 的重大技术转变，可能提升性能、安全性和生态系统兼容性，从而影响开发者采用率和 JavaScript 运行时市场的竞争格局。 重写大约在一周内完成，并依据一份详细的移植指南将 Zig 惯用法映射到 Rust 等效代码。代码库现在包含超过 10,000 个 unsafe 块，表明采用了机械翻译方法。

hackernews · Chaoses · May 14, 08:15 · [社区讨论](https://news.ycombinator.com/item?id=48132488)

**背景**: Bun 是一个快速的全能 JavaScript 运行时和工具包，最初用 Zig 编写。Rust 是一种以内存安全和性能著称的系统编程语言。此次重写旨在利用 Rust 的生态系统和工具链。

**社区讨论**: 社区评论讨论了移植指南和翻译的机械性，有人质疑为何不使用自动翻译工具。其他人注意到大量的 unsafe 块和代码库的复杂性，并将其与 Rust 编译器的大小进行比较。
