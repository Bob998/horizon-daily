## [F3：带 WASM 解码器的新型列式存储格式挑战 Parquet](https://github.com/future-file-format/f3) ⭐️ 5.0/10

F3 是一种提议的列式存储格式，它嵌入 WebAssembly（WASM）二进制文件来解码数据，旨在通过无需原生库的平台无关解码来改进 Apache Parquet。 如果被采用，F3 可以减少数据处理对特定语言 SDK 的依赖，可能降低集成成本。然而，其成功取决于能否克服 Parquet 的主导生态系统和兼容性优势。 每个 F3 文件包含数据和元数据，以及仅占用千字节的 WASM 解码器。该格式通过 Flatbuffers 描述，并在 GitHub 上开源，但目前缺乏广泛的工具支持以及与 Parquet 对比的明确性能基准。

hackernews · tosh · Jun 23, 16:53 · [社区讨论](https://news.ycombinator.com/item?id=48647799)

**背景**: Apache Parquet 是一种广泛采用的开源列式存储格式，针对分析优化，得到 Spark 和 Hive 等主要框架的支持。列式格式按列而非行存储数据，能够实现高效压缩和对大数据集的快速查询。WASM 是一种低级二进制指令格式，在虚拟机中运行，允许跨平台执行代码而无需重新编译。
