## [苹果 SpeechAnalyzer API 基准测试：速度更快但准确度略低](https://get-inscribe.com/blog/apple-speech-api-benchmark.html) ⭐️ 5.0/10

苹果新的 SpeechAnalyzer API 与 OpenAI 的 Whisper 及其前身进行了基准测试，结果显示速度显著提升但准确度略有下降。 这对在苹果平台上构建语音转文本应用的开发者很重要，因为它提供了更快的原生替代方案，但准确度的权衡可能影响需要高精度的使用场景。 基准测试在数学讲座上进行，SpeechAnalyzer 速度明显更快，仅比 Whisper-Large-V2 略差。该 API 专为实时转录设计，是苹果设备端语音识别框架的一部分。

hackernews · get-inscribe · Jul 13, 16:06 · [社区讨论](https://news.ycombinator.com/item?id=48894752)

**背景**: 语音识别 API 将音频转换为文本。苹果之前的 API 能力较弱，而 OpenAI 的 Whisper 是流行的开源模型，以高准确度但速度较慢著称。苹果的新 API 旨在提供更快、注重隐私的设备端替代方案。

**社区讨论**: 评论者指出 Whisper 已过时，建议与 Nvidia 的 Nemotron 或 Mistral 的 Voxtral 等新模型比较。一些人表示语音转文本正成为已解决的问题，质疑付费封装应用的长期可行性。

## [三星健康应用威胁：拒绝 AI 训练将删除用户数据](https://neow.in/cWsyMTV3) ⭐️ 5.0/10

三星健康应用计划收集用户的睡眠、用药、医疗记录和周期追踪等健康数据用于 AI 训练，并警告拒绝的用户将被删除相关数据。 该政策引发重大隐私担忧，可能削弱用户对三星健康生态的信任，进而影响三星可穿戴设备和健康服务的市场接受度。 目标数据类别包括睡眠模式、用药记录、医疗记录和经期追踪。拒绝 AI 训练的用户将面临健康数据被删除，导致许多设备功能无法使用。

hackernews · bundie · Jul 13, 20:01 · [社区讨论](https://news.ycombinator.com/item?id=48897991)

**背景**: 三星健康是与 Galaxy Watch 等三星可穿戴设备集成的健康追踪平台。科技公司利用用户数据训练 AI 日益普遍，但以删除数据作为强制同意的惩罚手段颇具争议，可能违反 GDPR 等数据保护法规。

**社区讨论**: 评论普遍持批评态度，用户认为该政策对用户不友好，并指出如果功能无法使用，三星应提供退款。有人讽刺地表示数据删除反而可视为隐私利好，也有人将其与谷歌的类似做法相比较。
