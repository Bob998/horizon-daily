## [使用 Git --author 标志阻止 GitHub 仓库中的 AI 机器人垃圾信息](https://archestra.ai/blog/only-responsible-ai) ⭐️ 5.0/10

一篇博客文章描述了如何使用 Git 的--author 标志来过滤 GitHub 仓库中的 AI 机器人垃圾信息，突显了针对日益严重的自动拉取请求和评论问题的变通方法。 这对面临来自 AI 生成贡献的日益增长的垃圾信息的开源维护者很重要，这些垃圾信息可能压垮项目并浪费维护者的时间。这也凸显了 GitHub 需要提供平台级解决方案来解决这个问题。 --author 标志允许按作者过滤提交，但这是一种客户端变通方法，并不能阻止垃圾信息被提交。文章指出 GitHub 当前的工具有限，社区呼吁增加如 PR 删除和贡献者声誉系统等功能。

hackernews · ildari · May 18, 15:24 · [社区讨论](https://news.ycombinator.com/item?id=48181125)

**背景**: GitHub 是一个流行的开源软件托管平台，任何人都可以提交拉取请求（PR）来贡献代码。最近，AI 机器人被用来生成低质量或垃圾 PR 和评论，使维护者不堪重负。Git 的--author 标志是一个命令行选项，用于按作者姓名或电子邮件过滤提交。

**社区讨论**: 评论者对 GitHub 没有实施基本的垃圾信息预防措施（如要求 PR 验证）表示失望。一些人建议使用基于声誉的系统或类似 ELO 的评分来过滤贡献，而另一些人则指责 AI 炒作鼓励了低质量的提交。

## [AI 生成的错误报告淹没 Linux 安全邮件列表](https://www.theregister.com/security/2026/05/18/linus-torvalds-says-ai-powered-bug-hunters-have-made-linux-security-mailing-list-almost-entirely-unmanageable/5241633) ⭐️ 5.0/10

Linus Torvalds 表示，AI 生成的错误报告使 Linux 安全邮件列表几乎无法管理，大量低质量的自动提交淹没了人工审查者。 这突显了依赖邮件列表进行安全报告的开源项目面临的日益严峻挑战，AI 生成的垃圾信息可能掩盖真实漏洞并减缓响应速度，从而影响广泛使用的软件的安全性。 Torvalds 的评论是在 Linux 7.1-rc4 发布期间发表的，他特别抱怨了一个文档变更。《The Register》的文章被批评为歪曲上下文并链接无关文章。

hackernews · jonbaer · May 18, 12:20 · [社区讨论](https://news.ycombinator.com/item?id=48178692)

**背景**: Linux 内核开发严重依赖邮件列表进行补丁提交和错误报告。像大型语言模型这样的 AI 工具可以生成看似合理但不正确的错误报告，增加噪音。另一位内核维护者 Greg Kroah-Hartman 最近指出，AI 对开源社区可能有用，这与 Torvalds 的沮丧形成对比。

**社区讨论**: 评论者指出，有人正在用大量 AI 生成的补丁垃圾邮件轰炸内核邮件列表，可能是为了毒化 LLM。有人建议，如果报告无法用两句话概括，应将其归类为垃圾邮件。其他人则质疑使用邮件列表进行开发的方式，更倾向于基于主题的平台。

**标签**: `#Linux`, `#AI`, `#security`, `#open source`, `#spam`
